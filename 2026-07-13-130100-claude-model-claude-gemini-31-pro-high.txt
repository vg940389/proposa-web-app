     ProposalApp — Full MVP Implementation Plan

     Context

     We are building a B2B SaaS web app for creating, signing, and collecting
     payment on business proposals (PandaDoc-style MVP). The project has a
     working scaffold with auth pages (Firebase Auth + Supabase bridge), a stub
     dashboard, 5 Supabase migrations, and TypeScript types — but no core
     features (proposal editor, public viewer, signature capture, payments).
     Zero git commits exist. The goal is to go from scaffold to deployed MVP
     across 4 milestones.

     ---
     Key Architectural Decisions

     - Auth: Keep Firebase Auth with existing Supabase bridge pattern (no
     migration)
     - Rich Text: TipTap WYSIWYG editor for proposal text sections
     - Images: File upload to Supabase Storage (not URL paste)
     - Drag & Drop: HTML5 Drag and Drop API (no library) + mobile up/down arrows
     - State: useReducer + Context for proposal editor (no external state
     library)
     - Signatures: Raw <canvas> with PointerEvents (no library)
     - Payments: Stripe Checkout via Supabase Edge Functions
